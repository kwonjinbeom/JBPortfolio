1. 프로젝트 기획서


제목 내용
기획 의도 포스트 코로나 시대로 들어서며 한동안 침체기를 겪었던 공연시장은 성장세로 진입하고 있다.

이전부터 공연문화는 '이선좌(이미 선점된 좌석입니다)'라는 신조어를 남길만큼 티켓팅 싸움이 치열하고 소비자들은 기존 공연예매사이트의 서버 및 관리 문제로 불만을 가지고 있다.

이에 본 프로젝트는 사용자에게 편리한 예매사이트를 서비스하는 것을 목표로 깔끔한 UI와 검색기능을 제공하고자 한다.
개발환경 - 개발 OS : Windows NT
- DBMS : ORACLE
- JDK 버전 : jdk-17
- 개발도구 : STS, SQL Developer
- WAS : Apache Tomcat 9.0
- 빌드 관리 : Maven
- 개발 언어 : Java, HTML5, CSS, JavaScript, jQuery, JSP, XML, MyBatis, Spring boot 등
- 형상 관리 : GitHub, Sourcetree
- API : KakaoMAP API, KakaoPAY API, import API
- 테스트 : Junit v4.12
- View Tool : Chrome, 익스플로러
요구사항 [예매 관리] - 권진범
사용자는 공연 별 예매창을 통해 공연 예매를 할 수 있다.
관리자는 공연 기간, 공연관, 상영시간, 좌석에 대한 예매 정보를 등록할 수 있다. 결제 내역을 확인할 수 있으며 결제 내역으로 월간, 주간 매출과 순이익, 공연별 매출 현황을 그래프로 확인할 수 있다.

[좌석 관리] - 권진범
사용자는 관리자가 등록한 정보를 바탕으로 활성화된 좌석을 선택할 수 있다.
관리자는 공연의 좌석, 좌석번호, 티켓 가격, 좌석 선택 여부에 대한 좌석 정보를 등록할 수 있다.

[결제 관리] - 권진범
사용자는 좌석을 선택한 후, 선택된 좌석의 가격에 맞게 결제 API(kakao pay, import)를 통해 결제를 할 수 있다.
